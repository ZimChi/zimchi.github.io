How I containerize dependency installs to stay safe from supply‑chain attacks.

Chielo Zimmerman

PROBLEM

Supply‑chain attacks target developer workstations by compromising the dependencies required to run applications under development.

Common dependencies in public package ecosystems involve many distributed contributors, making it easier for malicious actors to hide harmful code in legitimate changes or compromise a maintainer with publish access.

SOLUTION

Apply zero‑trust principles by installing dependencies from public package ecosystems inside Docker containers.

IMPLEMENTATION EXAMPLE

Dockerfile



FROM node:24-alpine

RUN apk add --no-cache bash shadow curl \
    && curl -sSL https://github.com/tianon/gosu/releases/download/1.17/gosu-amd64 \
         -o /usr/local/bin/gosu \
    && chmod +x /usr/local/bin/gosu

COPY normalize-user.sh /usr/local/bin/normalize-user.sh
RUN chmod +x /usr/local/bin/normalize-user.sh

WORKDIR /app

ENTRYPOINT ["/usr/local/bin/normalize-user.sh"]

normalize-user.sh - Place next to Dockerfile

#!/bin/sh
set -e

# Create an in‑container user matching the host's UID/GID
addgroup -g "$HOST_GID" containeruser
adduser -D -u "$HOST_UID" -G containeruser containeruser

# Open shell as new user
exec gosu containeruser /bin/sh

Run docker

docker buildx build -t express-app-image .
docker run -p 3000:3000 -it -v "$PWD":/app -e HOST_UID="$(id -u)" -e HOST_GID="$(id -g)" express-app-image

Do stuff within the Docker container shell.


npm install express-generator
npx express-generator my-app
cd my-app
npm install
npm start

Inspect with host browser

http://localhost:3000
